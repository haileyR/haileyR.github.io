CSS Positioning: Relative, Absolute, Fixed

2014. 09. 27

In CSS files, web elements are get styled with lots of css methods. For positioning them, there are 4 variables that can be used: static, relative, absolute, and fixed.

First, position: static; is the default value for web elements. If an element's position is not defined or defined as static, web elements will sit where they should be which means listing themselves in a webpage in the order of how they defined in the corresponding html file.

Relative: position: relative;

When an element is relatively positioned, the element can be moved with top, right, bottom and left values with respect to its original position. Its original place is still occupied for this element.

Absolute: position: absolute;

When an element is absolutely positioned, the element is placed relative to its first parent element. Here, it is important to know that the parent that the element is refering to should be other than static position and if there isn't any appropriate parent to refer to, it refers to block. It also uses top, right, bottom and left values to move the element, but in this case the element does not leave its original space reserved like relatively positioned and moved element.

Fixed: position: fixed;

When an element's position is fixed, it means that it always stays where it is positioned within the browser window. It does not move even when a page is scrolled down or up. The element does not take any other space other than it's current space and it can overlap other elements. In other words, other elements's position won't be affected by it.
